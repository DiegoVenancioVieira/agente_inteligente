Assistente de FAQ Municipal

Agente de IA que responde dúvidas dos municípes com base na legislação e nos documentos das secretarias — self-hosted, sem enviar dados a serviços externos.

💬 Chat 🔑 Painel 📖 Tutorial 📊 Relatório

Visão geral

O cidadão faz uma pergunta em linguagem natural; o sistema busca a resposta nos documentos oficiais da secretaria (RAG — busca semântica) e a devolve de forma curta e fundamentada. Se a informação não existir na base, ele recusa com honestidade em vez de inventar — comportamento essencial para um órgão público.

Um cache semântico guarda as respostas já dadas: perguntas repetidas voltam em ~0,3s sem acionar o modelo de IA, aliviando o servidor.

Arquitetura

Cidadão → interface de chat (/)
▼
Microserviço (VPS2, porta 8100) — API + cache semântico. Gera o embedding da pergunta, consulta o cache e orquestra a resposta.
▼  (se não estiver no cache)
AnythingLLM (VPS2, porta 3001) — faz a busca nos documentos (RAG) e monta o prompt.
▼
Ollama (VPS1, porta 11434) — modelos de IA em CPU: qwen2.5:7b (gera a resposta) e bge-m3 (embeddings, pt-BR).

Como uma pergunta é respondida

  • 1. Embedding — a pergunta vira um vetor (bge-m3).
  • 2. Cache — se houver pergunta quase idêntica já respondida (similaridade ≥ 0,85), devolve na hora.
  • 3. Busca (RAG) — senão, o AnythingLLM recupera os trechos mais relevantes dos documentos (corte de similaridade 0,55).
  • 4. Geração — o qwen2.5:7b redige a resposta apenas com base nos trechos; se nada for relevante, recusa.
  • 5. Cache — respostas fundamentadas são guardadas para as próximas vezes.

Segurança

  • Self-hosted / LGPD — nenhum dado do cidadão sai para serviços externos; tudo roda nas VPS da prefeitura.
  • Rate-limit — protege o modelo contra abuso (429 ao exceder o limite por IP).
  • Token de administração — o painel e a base de conhecimento só mudam com o ADMIN_TOKEN. Mantenha-o forte e secreto.

Manutenção

  • Atualizou um PDF? Limpe o cache (botão no painel ou POST /cache/clear) para não servir respostas antigas.
  • Invalidação automática — pode ser ligada via n8n (ver docs/n8n-invalidacao-cache.md no repositório).
  • Novas secretarias — cada uma ganha um workspace próprio no AnythingLLM; o serviço evolui para atender várias com um seletor de secretaria.
